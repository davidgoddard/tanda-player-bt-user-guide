Music Database – Creating the Library

Music is stored in two folders called “music” and “cortinas” on a USB memory stick.

If an empty memory stick is placed into the Tanda Player as the only device and it is re-started, then the Tanda Player will copy all the necessary files it needs for the database as well as creating these two folders for you.

Once the USB has the necessary files and folders created you can copy your own music to the corresponding folders.

The Cortinas can be full tracks or pre-cut-down songs. The folder can contain one level of additional folders and these can be called things like Pop or Jazz etc. as a way to indicate what they contain. The Tanda Player will use these folder names as sets of cortinas you can choose to accompany any playlist.  Note that the top folder MUST contain one song. This is so that if you choose cortinas for a playlist there is at least one to use.

The music folder can contain any number of folders and sub-folders etc. These can be copied from your main music library directly.

When the Tanda Player starts up it reads all music files and stores their full path and filename inside its own database files. If you subsequently add just one more file or even hundreds, it will spot these and add them without losing any information added previously. If you remove any files from the USB device these will be removed from the Tanda Player’s own database files next time it starts up.

Adding Music

After you have got the Tanda Player up and running, sooner or later you will want to add more music. Simply take the USB device to your main music library and copy the new files as required. Then return to the Tanda Player and re-start it. Upon startup the home page will show counters and these will indicate the progress in two tasks:

1.     Getting the basic information

2.     Getting detailed information.

The basic information is a quick scan which opens the file and reads the MP3 file tags for the artist and title of the song. It also notes the duration of the file as given by the operating system.

The detailed information is a more detailed scan of the file in which it is opened and verified to be a music file and is effectively played from start to finish. Provided it can be read the system determines the actual duration after removing any trailing silence there is in the song and also the maximum gain that could be applied to the song that would avoid any distortion.  This gain is then applied whenever the Tanda Player plays the song so as to ensure that all songs play with a consistent volume.

Removing Music

Any files that do not pass the detail scan are flagged and are viewable in the configuration / setup page under the “Music File Warnings & Errors” tab.

In this tab you can also see the songs that have been marked as to be hidden from the Tanda Player – because they are not suitable.

You can click on the button to delete these from the USB device to save some disk space either against each song one at a time or remove all of them in one go.

Alternatively you can simply use a PC to delete the files manually.

When the Tanda Player next starts up it scans the USB device and compares the list of files it finds with those it knew about previously and any that have gone are removed from its own database. Note too that any Tandas and/or Playlists that featured these songs are marked as invalid but are otherwise left where they are so that you can replace the files or tandas to restore the playlists etc.

Duplicated Songs

It is possible to have your music collection scanned by the Tanda Player as a way to attempt to spot multiple copies of the same song.
